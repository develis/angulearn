Pré-requisitos

Antes de partir para a instalação do Angular, é necessário baixar dois programas essenciais:

Angular-CLI

Você usa a CLI Angular para criar projetos, gerar código de aplicativo e biblioteca e executar uma variedade de tarefas de desenvolvimento em andamento, como teste, empacotamento e implantação. Para instalar o Angular CLI, abra uma janela de terminal e execute o seguinte comando:

npm install -g @angular/cli

Alguns comandos importantes

ng new
Solicita informações sobre os recursos a serem incluídos no aplicativo inicial. Aceite os padrões pressionando a tecla Enter ou Return.

ng serve --open
Inicia o servidor, observa seus arquivos e reconstrói o aplicativo conforme você faz alterações nesses arquivos.

ng g component nome
Cria um componente. Os componentes são o bloco de construção principal para aplicações angulares.

ng g service nome
Cria um service. São objetos instanciados apenas uma vez durante a vida útil de um aplicativo, contêm métodos que mantêm os dados ao longo dela.

ng g module nome
Cria um módulo. Um módulo é um mecanismo para agrupar componentes, diretivas, pipes e serviços que estão relacionados, de forma que possam ser combinados com outros módulos para criar uma aplicação.

Templates, diretivas, e data binding

Um modelo combina HTML com marcação angular que pode modificar elementos HTML antes de serem exibidos. As diretivas de modelo fornecem lógica de programa e a marcação de ligação conecta os dados do seu aplicativo e o DOM. Existem dois tipos de vinculação de dados:
A associação de eventos permite que seu aplicativo responda à entrada do usuário no ambiente de destino, atualizando os dados do aplicativo.
A vinculação de propriedades permite interpolar valores que são calculados a partir dos dados do seu aplicativo no HTML.
Antes de uma visualização ser exibida, o Angular avalia as diretivas e resolve a sintaxe de ligação no modelo para modificar os elementos HTML e o DOM, de acordo com os dados e a lógica do programa. O Angular oferece suporte a vinculação de dados bidirecional, o que significa que as alterações no DOM, como as escolhas do usuário, também são refletidas nos dados do programa.
Seus modelos podem usar canais para melhorar a experiência do usuário, transformando valores para exibição. Por exemplo, use barras verticais para exibir datas e valores monetários apropriados para a localidade de um usuário. O Angular fornece tubos predefinidos para transformações comuns e você também pode definir seus próprios pipes.
